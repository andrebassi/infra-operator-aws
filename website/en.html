Features

Everything you need for
Infrastructure as Code

Built with production workloads in mind, following Kubernetes best practices.

Kubernetes Native

Define AWS resources as Kubernetes CRDs. Use kubectl, Helm, ArgoCD, or any GitOps tool you already know.

Drift Detection

Automatically detect and optionally remediate configuration drift between your desired state and actual AWS resources.

Secure by Design

IRSA support, Kubernetes RBAC, and secure credential management. No credentials stored in manifests.

Prometheus Metrics

Built-in metrics for reconciliation counts, durations, errors, and drift detection. Ready for Grafana dashboards.

Clean Architecture

Built with Kubebuilder, following clean architecture principles. Extensible, maintainable, and well-tested.

Comprehensive Docs

Detailed documentation with examples for every resource type. Quick start guides and best practices included.

ComputeStack
All-in-One Infrastructure

Create complete infrastructure with a single resource. VPC, Subnets, Internet Gateway, Security Groups, and EC2 Bastion - all automatically provisioned.

  • Auto-generates SSH Key Pairs with Kubernetes Secrets
  • Cloud-Init support for EC2 initialization scripts
  • Auto-creates Subnets if not specified
  • Minimal configuration - just 5 lines of spec
computestack.yaml 
apiVersion: aws-infra-operator.runner.codes/v1alpha1
kind: ComputeStack
metadata:
  name: my-stack
spec:
  providerRef:
    name: aws-production
  vpcCIDR: "10.200.0.0/16"
  bastionInstance:
    enabled: true
    instanceType: t3.micro
    userData: |
      #!/bin/bash
      yum install -y docker
      systemctl start docker
Quick Start

Get Started in
Minutes

Install the operator and start managing AWS resources immediately.

1

Install with Helm

helm install infra-operator oci://ghcr.io/andrebassi/infra-operator \ --namespace infra-operator --create-namespace
2

Configure AWS Provider

kubectl apply -f - <<EOF apiVersion: aws-infra-operator.runner.codes/v1alpha1 kind: AWSProvider metadata: name: aws-production spec: region: us-east-1 credentialsSecret: name: aws-credentials EOF
3

Create Your First VPC

kubectl apply -f - <<EOF apiVersion: aws-infra-operator.runner.codes/v1alpha1 kind: VPC metadata: name: my-vpc spec: providerRef: name: aws-production cidrBlock: "10.0.0.0/16" enableDnsSupport: true EOF
