YAML Declarativo
vpc.yaml 
apiVersion: aws-infra-operator.runner.codes/v1alpha1
kind: VPC
metadata:
  name: production-vpc
spec:
  providerRef:

terminal 
# Kubernetes
$ kubectl apply -f vpc.yaml

# CLI (sem Kubernetes)
$ ./infra.operator apply -f vpc.yaml

ComputeStack
Infraestrutura Completa

Crie infraestrutura completa com um único recurso. VPC, Subnets, Internet Gateway, Security Groups e EC2 Bastion - tudo provisionado automaticamente.

  • Gera automaticamente Key Pairs SSH com Kubernetes Secrets
  • Suporte a Cloud-Init para scripts de inicialização EC2
  • Cria Subnets automaticamente se não especificadas
  • Configuração mínima - apenas 5 linhas de spec
computestack.yaml 
apiVersion: aws-infra-operator.runner.codes/v1alpha1
kind: ComputeStack
metadata:
  name: my-stack
spec:
  providerRef:
    name: aws-production
  vpcCIDR: "10.200.0.0/16"
  bastionInstance:
    enabled: true
    instanceType: t3.micro
    userData: |
      #!/bin/bash
      yum install -y docker
      systemctl start docker
Início Rápido

Comece em
Minutos

Instale o operador e comece a gerenciar recursos AWS imediatamente.

1

Instale com Helm

helm install infra-operator oci://ghcr.io/andrebassi/infra-operator \ --namespace infra-operator --create-namespace
2

Configure o AWS Provider

kubectl apply -f - <<EOF apiVersion: aws-infra-operator.runner.codes/v1alpha1 kind: AWSProvider metadata: name: aws-production spec: region: us-east-1 credentialsSecret: name: aws-credentials EOF
3

Crie Sua Primeira VPC

kubectl apply -f - <<EOF apiVersion: aws-infra-operator.runner.codes/v1alpha1 kind: VPC metadata: name: my-vpc spec: providerRef: name: aws-production cidrBlock: "10.0.0.0/16" enableDnsSupport: true EOF
